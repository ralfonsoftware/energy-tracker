## Deferred from: code review of localDate.ts extraction (2026-07-03)

- `addMonths`'s `setMonth()` arithmetic overflows on month-end boundaries — e.g. Jan 31 + 1 month lands on Mar 3, not Feb 28, because JS `Date` rolls excess days into the following month. - `localDate.ts` has no `NaN`/`Invalid Date` guarding anywhere. - Round-trip asymmetry between `TariffForm.tsx`'s create-flow write path (`contractStartDate: \`${data.contractStartDate}T00:00:00Z\``, unchanged by this extraction) and `parseLocalDate`'s local-calendar-date read path: a user west of UTC who picks "today" and later views/edits that tariff can see the date rendered one calendar day earlier than what they entered, because the write path always encodes literal UTC midnight while the read path extracts local calendar-date parts from that UTC instant.
