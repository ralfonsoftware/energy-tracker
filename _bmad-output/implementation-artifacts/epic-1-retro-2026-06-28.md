## Challenges & Learnings

### 1. SWA deployment correctness can't be verified from code alone
Story 1.4 was marked "done" with correct code, but required 4 post-deployment git commits:
- Removed SWA Free tier `auth` block (not supported on Free SKU)
- Wired Entra ID app registration via Portal instead
- Fixed unsubstituted `{request.pathAndQuery}` in 401 redirect
- Fixed deploy path from `client/dist` instead of `client/`

**Root cause:** "Done" was defined as code-correct, not deployed-verified. Infrastructure stories have a deployment correctness dimension that code review cannot catch.

### 2. Ecosystem surprises in every story
Unforeseeable dependency issues hit in 4 of 5 stories:
- FluentAssertions v8 switched to commercial license (1.1) → replaced with Shouldly
- xunit runner v3 incompatible with xunit v2 test package (1.1)
- TypeScript 6.0 `baseUrl` deprecation (1.1)
- `ConfigureFunctionsWebApplication` worker overload not matching Azure docs (1.4)
- SWA Free tier auth block limitation undocumented (1.2/1.4)

**Root cause:** These were caught in code review rather than during implementation planning.

### 3. UX pre-review gap on frontend stories
Story 1.5 had 5 design-level issues reach code review:
- Duplicate `aria-label="Main navigation"` on two nav landmarks
- `text-micro` missing `text-transform: uppercase`
- Active sidebar `borderRadius` hardcoded as `'10px'` instead of CSS var
- Missing `-webkit-backdrop-filter` vendor prefix
- `accent-over-budget` and `accent-spike` sharing identical hex `#f59e0b`

**Root cause:** No self-check against `DESIGN.md` tokens before marking ready for review.

### 4. No CI test gate (for the entire epic)
18 tests existed with no CI step running them. Every merge to `main` ran zero tests.

## Actions Taken During This Retrospective

### ✅ Fixed: `accent-over-budget` color token
- **Before:** `#f59e0b` (same as `accent-spike`)
- **After:** `#fb923c` (orange-400)
- **Rationale:** Spike = data observation (amber), over-budget = financial alarm (orange), error = system failure (coral `#f87171`). Three visually distinct states.
- **File:** `client/src/index.css`

### ✅ Fixed: CI test gate
Added two steps to `.github/workflows/azure-static-web-apps.yml`:
- **"Test frontend":** `npm ci && npm test -- --run` (vitest, blocks on failure)
- **"Test backend":** `dotnet test api.Tests/` (xUnit, blocks on failure)

Pipeline order is now: test frontend → build frontend → test backend → publish backend → migrate → deploy. A broken test blocks all subsequent steps.

## Key Lessons for Future Epics

1. **Infrastructure stories need a deployed verification step in DoD** — code review alone cannot catch SKU limitations or config format restrictions.
2. **Check new dependencies for license, version compatibility, and known breaking changes before implementation** — not in code review.
3. **Frontend stories: cross-check all CSS values and token names against DESIGN.md before review** — the design system is the source of truth, not intuition.
4. **Keep tests wired to CI from day one** — even a scaffold with one passing test should have a CI gate.
5. **Dev Notes format established in Story 1.1 is working** — maintain it in every story.
