## What Went Well

1. **The design-gate pattern (established in Story 8.4) continued cleanly.** Stories 9.1 (D-45, save-affordance), 9.4 (D-44 amendment, Smart Power Strip formula), and 9.6 (D-46, ghost-card treatment) each formalized an approved decision before any code — no rework, no ambiguity for the implementing story.
2. **Verify-before-implement caught real waste before it happened.** Story 9.3's live comparison found the room-list Save buttons already matched the target visual pattern byte-for-byte — zero source changes shipped. Story 9.1 was a pure spec-confirmation gate. Two stories, zero wasted implementation work.
3. **The Rescope Note pattern turned stale backlog items into honest, narrow scope instead of wasted or misdirected work.** Stories 9.7, 9.9, and 9.11 each found their originating `deferred-work.md` premise had already been resolved by earlier work (Story 6.0's precision pass, Story 9.7's own refactor) and rescoped to the real remaining gap (DRY refactor, explicit-null rejection, a test-coverage gap) rather than "fixing" a bug that no longer existed.
4. **Story 9.12 is the standout catch of the epic.** Code review didn't just find a code bug — it found circular reasoning in the dev agent's own investigation doc (the "one flat confirmed unaffected" claim was self-referential, comparing post-migration state to itself). The story was corrected to state plainly that no flat is actually cleared, and closed as an honest "accepted risk, permanently unauditable" rather than overclaiming a clean result.
5. **Adversarial code review kept finding real, non-cosmetic issues epic-wide** — zero-height chart bars hiding the entire point of Story 9.8's fix; a wrong Bicep line citation and a mislabeled "second limitation" claim in 9.12; a missing frontend retry-on-409 path in 9.10 that would have looped a genuinely conflicted user forever; a stale boolean-AND bug in `SettingsPage.tsx` that could never surface its own error banner. The review layer is earning its keep.
6. **Story 9.10's full-scope concurrency hardening was the right call, not scope creep.** What started as "add RowVersion to 6 entities" surfaced ~10 additional Functions/DTOs needing response-shape parity once the team refused to ship a half-protected write path. Confirmed by Ralf this retro: doing the complete cross-cutting fix in one pass beat splitting or deferring it.

## Challenges & Patterns

1. **`deferred-work.md` items go stale before anyone re-checks them.** Four stories this epic (9.7, 9.9, 9.11, and 9.12's origin) discovered their premise no longer held, only at epic-planning/story-creation time. Story 9.12 is the sharpest case: the SQL PITR audit window had *already closed* (Basic tier, 7-day max, 16 days elapsed) before the story even started — a real, permanent loss of an option that a tighter check cadence might have caught in time to act on.
2. **Decision/investigation-doc sprawl is diluting the knowledge baseline.** `.decision-log.md` (D-44, D-45, D-46...) and the `investigations/` folder keep growing, but settled decisions are never folded back into `architecture.md`/PRD. A new story author has to know to go hunting across several addenda to get the current picture.
3. **Local dev environment standup has recurring friction.** Story 9.2 alone lost time to Azurite not running, a Blob Storage connection-refused error, a SQL Server firewall block requiring Ralf's manual intervention, and a stale orphaned SWA CLI process on port 4280. None of this was code work — all setup tax, paid repeatedly across sessions.
4. **`TariffResolver.ResolveAsync` is documented as dead code but was never actually removed.** Flagged since Epic 7's retro and reflagged in `project-context.md`'s invariants section, it still sits in the codebase (`api/Shared/TariffResolver.cs`) with its own DI registration and dedicated test file — a landmine Epic 10's tariff-aware detectors could trip over if not cleaned up first.
